# 猫糖尿病治疗分支图 / Feline Diabetes Treatment Branch Map

## 本页回答的问题 / Question This Page Answers

在证据足够强以进行排名之前，应如何分离治疗选项？

How should treatment options be separated before the evidence is strong enough for ranking?

## 核心要点 / Core Takeaway

当前糖尿病治疗图没有安全的单一赢家。正确的结构是按角色、证据类型、候选选择、终点和安全边界进行分支分离。

The current diabetes treatment map has no safe single winner. The right structure is branch separation by role, evidence type, candidate selection, endpoint, and safety boundary.

## 分支比较 / Branch Comparison

| 分支 Branch | 当前角色 | 最强安全声明 | 主要边界 |
|---|---|---|---|
| 饮食架构 Diet architecture | 基础管理 | 饮食影响血糖和胰岛素需求终点 | 非通用处方 |
| 胰岛素/甘精 U300 | 核心药理分支 | 有前景的短期甘精 U300 信号 | 无比较优越性 |
| SGLT2 抑制剂 | 美国 FDA 监管口服产品分支 | 为选定的其他方面健康胰岛素幼稚猫批准的选项 | 酮症酸中毒风险和候选选择 |
| α-葡萄糖苷酶抑制剂辅助 | 历史辅助加饮食分支 | 组合干预信号 | 独立药物效应未隔离 |
| 肠促胰素/GLP-1/DPP-4 前沿 | 创新分支 | 合理的代谢机制 | 非常规方案权威 |
| 缓解 Remission | 终点/控制分支 | 真实但证据脆弱的终点 | 无单一预测因子或方案 |

## 护栏 / Guardrail

不要从本层按优越性排名治疗。不要将口服途径推广为本质上更好。不要将 Bexacat 和 Senvelgo 作为赢家比较。不要仅从当前编译层编写面向主人的临床说明。

Do not rank treatments by superiority from this layer. Do not promote oral route as inherently better. Do not compare Bexacat and Senvelgo as winners. Do not write owner-facing clinical instructions from the current compiled layer alone.
