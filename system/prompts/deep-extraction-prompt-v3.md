# Deep Extraction Prompt V3

## 定位

V3 深度提炼产出独立的提炼文档，用于：
1. **二级页面展示**：研究者可以直接阅读 Phase 2 + Phase 3，无需读原文
2. **一级页面摘要**：从 Phase 2/3 中提炼 1-2 句话用于推荐卡片
3. **原文入口**：提供外链，研究者可选择性验证

这不是 source card 字段更新工具，而是产出完整的深度阅读替代物。

## 处理流程

### Step 1: 获取原文
- 优先使用 PDF 全文
- 如无法获取，使用摘要 + 可访问的图表/表格
- 在文档开头注明数据来源和局限

### Step 2: Phase 0 逐段分析
- 按论文原始结构逐段处理
- 不要跳过任何重要段落
- 捕获所有硬信息：数字、日期、样本量、p 值、产品名、组织名

### Step 3: Phase 1 主题重构
- 从 Phase 0 中归纳自然主题
- 不强行套用固定分类
- 保留所有硬信息

### Step 4: Phase 2 论点-论据提炼
- 每个主题提炼 2-4 个核心论点
- 每个论点必须有：论据、细节、隐含前提
- 这是研究者在二级页面阅读的主要内容

### Step 5: Phase 3 自检
- 最重要的发现（不是标题说的，而是真正的贡献）
- 容易被误读的点
- 证据边界（这篇论文不能证明什么）
- 对特定应用的启发

### Step 6: 一句话总结
- 从 Phase 2/3 中提炼
- 用于一级页面 Takeaway
- 必须是具体的、有信息量的
- 不能是 "这是一篇关于 X 的重要研究"

---

## 质量标准

### Phase 2 论点
- ✅ 是完整主张，不是话题描述
- ✅ 有具体论据支持
- ✅ 明确隐含前提
- ❌ "本文研究了..."
- ❌ "这是一个重要的发现"

### Phase 3 自检
- ✅ 指出具体的证据边界（研究设计层面）
- ✅ 识别容易被误读的点
- ❌ "需要更多研究"
- ❌ 通用的学术套话

### 一句话总结
- ✅ 具体、有信息量
- ✅ 从 Phase 2/3 内容中自然导出
- ❌ 重复标题
- ❌ 空泛赞美

## 与 Source Card 的关系

深度提炼文档是独立文件，但可以反向更新 source card：

```yaml
# 在 source card 中添加
deep_extraction:
  file: "raw/deep-extractions/ext-{source-id}.md"
  one_line_summary: "[从 Phase 2/3 提炼的一句话]"
  why_it_matters: "[从 Phase 2/3 提炼的价值陈述]"
```

## 适用场景

### 适合深度提炼
- 高证据等级论文（指南、系统综述、Meta 分析）
- 原创研究且有详细方法和数据
- 与现有 vault 内容有张力或延伸的论文
- 2020 年后的主要临床研究

### 不适合深度提炼
- 纯综述无新数据
- 标题即全部内容的短报告
- 已被更新版本取代的旧指南
- 只需 title-only 收录的低优先级来源
