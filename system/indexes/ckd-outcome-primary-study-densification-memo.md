# CKD Outcome Primary-Study Densification Memo

- Date: `2026-04-09`
- Scope: current CKD vault after the outcome-architecture layer was established

This memo exists to answer one practical question:

If the vault now has a clearer outcome architecture, which primary-study cluster would most improve it next?

## Core Position

1. The next gains should come from primary-study densification, not more framework pages.
2. The highest-value studies are the ones that make outcome architecture more concrete, not the ones that merely add another plausible branch.
3. The current best densification cluster sits at the junction of:
   - pathology-linked endpoints
   - early-detection / biomarker signal
   - treatment and supportive-care consequence
4. The goal is not “read more papers.”
   The goal is “reduce outcome ambiguity in the compiled pages.”

## What This Memo Optimizes For

The next primary-study cluster should improve one or more of these:

- clearer separation between routine endpoints and trial outcomes
- stronger linkage from endpoint movement to patient-level consequence
- better grounding for supportive-care and burden outcomes
- fewer broad claims resting only on review-level compression

## What This Cluster Would Change If Done Well

1. The vault could stop speaking about “outcomes” in one undifferentiated way.
2. Supportive-care branches would connect more cleanly to burden outcomes.
3. Early-detection frontier claims would sit more explicitly inside the outcome architecture.
4. Endpoint movement would connect more clearly to pathology burden and patient consequence.

## What Not To Do Next

- do not add another route memo before this cluster is denser
- do not keep adding new framework pages without extracting more primary-study detail
- do not let machine-learning novelty or supportive-care familiarity outrun outcome-grounding
