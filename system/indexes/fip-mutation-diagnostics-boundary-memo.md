# FIP Mutation-Diagnostics Boundary Memo

- Date: `2026-04-09`
- Scope: `src-fip-009`, `src-fip-010`, `src-fip-018`, `src-fip-022`

This memo exists to stop the mutation branch from collapsing into one confused object.

Right now the vault has both:

- papers about mutation-origin and systemic emergence
- papers about mutation detection as a diagnostic tool
- papers about limitations of that diagnostic use

Those are not the same thing.

## Core Position

1. Mutation-origin belongs to the mechanism layer first.
2. Mutation-detection belongs to the diagnostic-support layer.
3. Mutation-based diagnosis is real enough to matter, but too fragile to be treated as a certainty engine.
4. The current vault should talk about `boundary`, not `resolution`, in this branch.

## Three Different Questions

### Question 1: How does FIP emerge?

This is where `src-fip-009` belongs.

The right output here is:

- pathobiogenesis
- systemic emergence
- disease-from-background logic

Not:

- definitive clinical diagnosis

### Question 2: Can mutation-related assays help diagnosis?

This is where `src-fip-022` belongs.

The right output here is:

- bounded utility
- sample- and context-sensitive support

Not:

- one-step certainty

### Question 3: What are the limits of mutation-based diagnosis?

This is where `src-fip-010` and `src-fip-018` belong.

The right output here is:

- caution
- boundary
- anti-overclaim logic

Not:

- dismissal of the whole branch

## Current Working Hierarchy

| Mutation Branch | Current Position In Vault | Notes |
|---|---|---|
| Mutation-origin mechanism logic | strongest current mutation anchor | explains emergence, not diagnosis |
| Systemic-spread correlation logic | intermediate boundary layer | explains what mutation-associated signals may be tracking before overt disease certainty |
| Mutation-detection utility | real but bounded supportive branch | belongs in workup, below certainty |
| Mutation-diagnostic limitation | essential caution branch | prevents overclaim and flattening |

## What The Vault Can Now Say More Clearly

1. The mutation branch is not one argument; it is a three-part architecture.
2. Origin evidence should not be promoted into test certainty.
3. Diagnostic mutation assays should not be erased just because they have limitations.
4. The right language here is bounded supportive value plus explicit overclaim resistance.
5. The utility side of mutation testing is now better represented, but still only as comparative bounded support.
6. The limitation side now has a real deep-extraction anchor rather than a title-level placeholder.
