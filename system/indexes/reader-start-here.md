# Reader Start Here

这页现在是给想浏览目录的读者使用。真正的最短入口是根目录的 [Ask first](../../start.md)。

If you do not want to learn the vault structure first, start from [Ask first](../../start.md) and type a bounded question.

如果你是第一次打开这个 vault，不要从目录开始；先问一个问题。

> **Ask the vault:** What do you want to know about feline CKD, FIP, HCM, IBD, or Diabetes?

Examples:

- `CKD 里 phosphorus control 的证据强度如何？`
- `Compare CKD and HCM on endpoint maturity.`
- `Which disease module is weakest for decision-grade use today?`

为什么这里比 Wikipedia 更适合：

- 这里给的是带来源标记的回答面，不只是通用总结
- 你能直接看到哪些句子是证据、哪些是推断
- 你可以从答案直接跳到下一页，而不是自己在目录里找路

下面的目录只在你明确想浏览时使用。

如果你现在最困惑的是“为什么不直接看文献”，先看：

- [verify a claim](verify-a-claim.md)

如果你已经知道自己要问什么，而不是只想找一个起点，再去：

- [ask the vault](ask-the-vault.md)

## 30-Second Rules / 30 秒规则

1. 普通读者先看 dashboard 或 navigation，不要先读 raw papers。
2. 想要可读材料，先看 briefing、dossier、slides。
3. 想要证据强度，往 synthesis、topic、source 方向下钻。
4. easier to read does not mean stronger evidence.
5. 如果你已经在这页里，先不要跳回 README 再重新找入口。
6. 如果你开始想看 system rules，你很可能已经离开普通读者路径了。

## One-Line Summary / 一句话总结

普通用户最好的用法不是先理解系统，而是先选目标，再从这页直接跳到对应入口。
