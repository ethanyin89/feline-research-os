# FIP Treatment Evidence Memo

- Date: `2026-04-09`
- Scope: `src-fip-003`, `src-fip-013`, `src-fip-016`, `src-fip-017`, `src-fip-019`, `src-fip-024`

This memo exists to prevent the current FIP treatment branch from being flattened into one vague sentence like:

`GS works`

That sentence is directionally important, but it is too flat for this vault.

## Core Position

1. The current FIP treatment branch is real and much stronger than the current regulatory branch.
2. The branch is not flat.
3. Different treatment papers are carrying different treatment objects:
   - baseline antiviral efficacy
   - real-world treatment package logic
   - neurologic rescue / high-complexity treatment
   - remission durability
4. The safest current language is `treatment transformation`, not `treatment question solved`.
5. The experimental antiviral foundation should sit below, not inside, the natural-disease baseline layer.

## Current Treatment Layers

### Layer 1: Baseline Antiviral Efficacy

This layer is mainly carried by:

- `src-fip-016`
- `src-fip-017`

What it means:

- antiviral activity is not theoretical only
- naturally occurring disease response exists
- the pre-antiviral fatalism frame is no longer sufficient
- the experimental foundation and natural-disease baseline are adjacent but not interchangeable

### Layer 2: Real-World Treatment Package Logic

This layer is mainly carried by:

- `src-fip-019`

What it means:

- practical treatment has already moved beyond one clean baseline study
- mixed-form disease handling and protocol packages matter
- real-world effectiveness and definitive hierarchy proof are different things

### Layer 3: Neurologic Rescue / High-Complexity Treatment

This layer is mainly carried by:

- `src-fip-024`

What it means:

- neurologic FIP is not just baseline treatment with a slightly worse prognosis
- dose, monitoring, and relapse logic become meaningfully different

### Layer 4: Remission Durability

This layer is mainly carried by:

- `src-fip-013`

What it means:

- the branch is not only about initial response
- longer-term remission and post-treatment status now matter inside the module
- no confirmed relapse in a selected follow-up cohort is a strong durability signal, but not a whole-branch cure estimate

## What The Current Sources Actually Support

### Stronger Support

- modern antiviral treatment changed the translational meaning of FIP
- GS-441524 baseline efficacy in naturally occurring disease is real
- experimental antiviral support is strong enough to justify a true preclinical foundation layer
- treatment-package logic exists in real-world case-series evidence
- neurologic FIP can no longer be treated as automatically outside the treatment branch
- post-treatment durability now has a real follow-up anchor rather than a placeholder label

### Bounded Support

- broad protocol hierarchy is not yet settled
- access pathway and regulatory legitimacy are not solved by efficacy papers
- package success does not erase retrospective-design limitations
- neurologic rescue remains a high-complexity branch with very small-sample evidence
- experimental rescue does not replace natural-disease clinical hierarchy
- remission durability evidence comes from a success-selected cohort and should not be reused as broad efficacy denominator

## What This Branch Is Not Allowed To Become

- It is not allowed to become one generic `GS works` sentence.
- It is not allowed to flatten baseline efficacy, package logic, and neurologic rescue into the same evidence tier.
- It is not allowed to confuse real clinical transformation with final optimization of protocol.
- It is not allowed to let treatment enthusiasm erase diagnostic and implementation complexity.

## What The Vault Can Now Say More Clearly

1. FIP treatment has genuinely changed, and that change is now one of the central facts of the module.
2. The treatment branch has at least four distinct operational layers: experimental foundation, baseline efficacy, package logic, and neurologic rescue.
3. Remission durability is now part of the branch, not a peripheral anecdote.
4. The branch is strong enough for a real translational layer, but not yet flat enough for simplistic ranking language.
