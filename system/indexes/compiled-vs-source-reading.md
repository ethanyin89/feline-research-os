# Compiled Pages vs Original Papers

这页回答的是：

`为什么不直接只看文献？什么时候应该同时看 compiled page 和原文？`

## Short Answer / 最短答案

**EN**
Compiled pages and original papers do different jobs.

- compiled pages are for orientation, compression, and cross-paper structure
- original papers are for verification, nuance, and exact wording

The best real usage is often both together, not one replacing the other.

**ZH**
compiled page 和原文献做的不是同一件事。

- compiled page 负责定向、压缩、跨文献结构化
- 原文献负责核实、细节、精确表述

最真实的使用方式，往往是两者一起看，不是谁替代谁。

## What You Get From Compiled Pages / compiled page 提供什么

**EN**
- the big picture without reading 10 to 30 papers first
- the current branch structure, for example mechanism, recognition, endpoint, translation
- clearer boundaries, for example support marker vs frontier marker, IBD vs lymphoma
- faster entry into what is already strong versus still thin

**ZH**
- 不用先读 10 到 30 篇文献，就能先拿到大图
- 先看到当前分支结构，比如 mechanism、recognition、endpoint、translation
- 更容易看清边界，比如 support marker 和 frontier marker，IBD 和 lymphoma
- 更快知道哪些结论已经较稳，哪些地方还薄

## What You Get From Original Papers / 原文献提供什么

**EN**
- the exact study design
- the exact population, sample size, and limitation
- the exact wording of claims
- the evidence you need when a conclusion feels too compressed or too smooth

**ZH**
- 具体研究设计
- 具体样本、对象和局限
- 具体 claim 原话
- 当某个结论看起来被压得太平滑时，你需要回去看的证据本体

## The Real Difference / 真正的区别

Compiled pages answer:

- `what is the current structured reading of this disease area?`

Original papers answer:

- `did this specific study really show that? under what conditions?`

That is the whole game.

## When You Should Read Compiled First / 什么时候先看 compiled

先看 compiled 更合适，如果你要做的是：

1. 刚进入一个病种，还没有大图
2. 想先知道现在这个 vault 已经整理到哪一步
3. 想快速区分 core spine 和 edge branch
4. 想先决定下一步该钻哪条线
5. 想拿一份更适合讨论的材料，而不是直接陷进文献细节

推荐入口：

- [reader start here](reader-start-here.md)
- [ordinary user usage guide](ordinary-user-usage-guide.md)
- disease `dashboard`
- disease `navigation`
- `synthesis-index`

## When You Should Go Back To Papers / 什么时候一定要回原文

一定要回原文，如果你要做的是：

1. 判断一个结论到底有多稳
2. 想核对 sample size、inclusion、methods、limitations
3. 想确认某个 marker 或 treatment 到底是 support 还是 lead evidence
4. 想知道 compiled page 有没有压掉关键例外
5. 要写更严肃的分析，而不是内部快速总结

推荐入口：

- `raw/papers/`
- source cards
- deep extraction worksheet

## Best Actual Workflow / 最实用的双轨工作流

最实用的用法不是二选一，而是这 5 步：

1. 先看 `dashboard` 或 `navigation`
2. 再看 `briefing` 或 `synthesis-index`
3. 锁定你真正关心的 branch 或 claim
4. 回到对应 source card 或原文献核实
5. 再回来读 compiled page，看这个 claim 在整个疾病结构里放在哪

也就是：

`compiled 先定向，paper 再核实，最后回 compiled 放回结构里`

## Common Mistake / 最常见误区

最常见的误区有两个：

1. 只看 compiled，然后把压缩后的说法当成原始证据
2. 只看原文，然后自己重新手工拼整个领域结构

前者会过度相信摘要层。

后者会浪费大量时间，而且很容易在 branch boundary 上走偏。

## One-Line Rule / 一句话规则

If you want orientation, start with compiled.

If you want proof, go to the paper.

If you want to work well, use both.
