# FIP Effusion Recognition Memo

- Date: `2026-04-09`
- Scope: `src-fip-003`, `src-fip-006`, `src-fip-007`, `src-fip-015`

This memo exists to stop the recognition branch from talking about `wet FIP` as though it were only a casual subtype label.

## Core Position

1. Effusion-focused recognition is one of the most operationally useful branches in FIP suspicion.
2. Effusive and non-effusive disease should not be collapsed into one flat presentation.
3. Effusion belongs in the suspicion-pattern layer first, not as a late confirmatory afterthought.
4. Effusion-guided suspicion is strong, but still does not replace composite diagnosis.
5. Effusive disease also has a bounded immunopathology layer and should not be reduced to fluid pattern alone.

## Current Effusion Recognition Layers

### Layer 1: Broad Disease-Form Framing

This layer is mainly carried by:

- `src-fip-003`

What it means:

- the wet/dry split remains fundamental to how FIP is recognized
- disease-form language belongs near the start of the workup

### Layer 2: Practical Clinicopathologic Pattern Recognition

This layer is mainly carried by:

- `src-fip-006`

What it means:

- effusion helps clinicians enter high-suspicion mode quickly
- practical case-pattern recognition matters before molecular strengthening

### Layer 2.5: Effusive Immunopathology Context

This layer is mainly carried by:

- `src-fip-007`

What it means:

- effusive disease is not only a recognition shortcut
- older immunology work adds disease-form-specific biological context beneath the fluid pattern
- this layer should deepen interpretation without outranking clinicopathologic suspicion

### Layer 3: Structured Disease-Form Recognition

This layer is mainly carried by:

- `src-fip-015`

What it means:

- wet versus dry recognition should be modeled more explicitly
- staging and disease-form description can sharpen how effusion is interpreted inside broader clinicopathology

## Current Working Hierarchy

| Recognition Branch | Current Position In Vault | Notes |
|---|---|---|
| Effusion-guided suspicion | strongest practical disease-form trigger | highly operational, but not definitive |
| Effusive immunopathology context | bounded background branch | deepens interpretation without leading workup |
| Broader clinicopathology and staging | leading structured recognition layer | helps keep effusion inside a fuller pattern |
| Neurologic extension | separate branch shift | should not be confused with fluid-led ordinary recognition |
| Bounded molecular strengthening | downstream support | should follow, not lead, suspicion pattern |

## What The Current Sources Actually Support

### Stronger Support

- effusion should be treated as a high-value recognition feature
- disease-form-aware recognition is more useful than generic `FIP looks variable` language
- clinicopathology gives effusion meaning inside a broader suspicion pattern
- older effusive-form immunology helps keep the branch biologically richer than a simple fluid-description shortcut

### Bounded Support

- effusion alone is not definitive diagnosis
- non-effusive disease still matters and should not disappear from recognition architecture
- effusion-led suspicion must still remain inside composite-support diagnosis

## What This Branch Is Not Allowed To Become

- It is not allowed to become a casual `wet vs dry` label with no operational meaning.
- It is not allowed to reduce recognition to `there is fluid, therefore FIP`.
- It is not allowed to erase non-effusive and neurologic complexity.
- It is not allowed to outrun the broader clinicopathology layer that gives it context.

## What The Vault Can Now Say More Clearly

1. Effusion is one of the cleanest practical recognition triggers in FIP.
2. Effusion belongs at the front of suspicion-pattern recognition, not buried inside generic case description.
3. The right framing is `effusion-guided suspicion inside clinicopathology`, not `effusion equals diagnosis`.
4. Non-effusive and neurologic forms still stop the module from turning fluid findings into certainty.
