- broad disease architecture and control logic

## Why This Order

- start with review and guideline anchors so FCV does not collapse into one vaccine or one symptom story
- lock the vaccination and persistence boundary early, because FCV control claims are easy to overstate
- add epidemiology before narrow extension branches, so recognition stays population-aware
- read ocular and enteric papers after the main respiratory/oral frame is stable

## Family Logic Behind This Order

- Tier A is heavy on `guideline + broad review` because those sources are best at setting disease architecture, not because they are automatically the highest evidence for every claim
- Tier B is where the strongest `original-study` anchors sit, because that is where endpoint ownership, challenge detail, and field-recognition detail become sharper
- Tier C contains useful but more claim-sensitive papers, including therapy frontier, extension branches, and context-only material

## Claim-Fit Reminders

- use `src-fcv-004` to control practical diagnosis/management order, not product winner logic
- use `src-fcv-015` to control mechanism compression, not present-day prevalence claims
- use `src-fcv-006` to control recognition caution, not universal prevalence
- use `src-fcv-017` and `src-fcv-022` to separate persistence and cellular-immunity logic, not to crown a vaccine winner
- keep `src-fcv-019` low as commentary/context only
