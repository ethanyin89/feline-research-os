# FIP Antiviral Product-Archetype Route Boundary Memo

Date: `2026-04-21`

Scope: first-pass product-archetype boundary work for later FIP antiviral regulatory route analysis.

This memo is not a jurisdiction recommendation.

It does one narrower thing:

`separate the FIP antiviral product objects before anyone tries to choose a route`

## Core Takeaway

`FIP antiviral route work should not start with jurisdiction. It should start with product-archetype separation.`

The current FIP treatment branch is strong enough to support route-boundary work.

It is not strong enough to support final route advice.

## Why This Memo Exists Now

The regulatory branch already has a first jurisdiction split across China, USA, EU, and UK.

The missing layer is product-object discipline.

Without that layer, the phrase `FIP antiviral` collapses at least three different things:

1. baseline GS-441524 natural-disease treatment
2. remdesivir plus GS-441524 real-world package logic
3. neurologic / high-complexity treatment branch

Those are not interchangeable regulatory objects.

## Product Archetype Set

### Archetype 1: Baseline GS-441524 Antiviral Product

Best current anchors:

- [src-fip-016 deep extraction round 1](src-fip-016-deep-extraction-round1.md)
- [src-fip-017 deep extraction round 1](src-fip-017-deep-extraction-round1.md)

Current meaning:

- this is the strongest current treatment-facing archetype
- natural-disease efficacy exists, and the experimental foundation is real
- the claim must stay bounded by case mix because severe neurologic and ocular FIP were not recruited in the baseline natural-disease study

Claim-fit:

- treatment of naturally occurring FIP
- baseline antiviral efficacy
- defined course and response-management framing

Claim style to avoid:

- universal all-form FIP treatment claim
- cure language that erases early deaths, relapse, retreatment, or case-mix limits
- regulatory legitimacy language based only on treatment efficacy

Current internal read:

- strongest first product archetype for regulatory stress testing
- still requires jurisdiction-specific eligibility and data-package analysis

### Archetype 2: Remdesivir Plus GS-441524 Treatment Package

Best current anchor:

- [src-fip-019 deep extraction round 1](src-fip-019-deep-extraction-round1.md)

Current meaning:

- this is a practice-like package archetype, not just a second copy of the baseline GS branch
- the evidence has broader practical shape, including mixed disease forms and ocular / neurologic involvement
- the design is still retrospective case-series evidence, so it should not become final hierarchy proof

Claim-fit:

- real-world treatment package
- practical response and remission framing
- mixed-form disease handling

Claim style to avoid:

- definitive superiority over baseline GS
- protocol-ranking language
- treating retrospective breadth as controlled comparative proof

Current internal read:

- high operational relevance
- weaker route cleanliness than a single-active baseline product, because product identity and package logic need to be defined first

### Archetype 3: Neurologic / High-Complexity Antiviral Branch

Best current anchor:

- [src-fip-024 deep extraction round 1](src-fip-024-deep-extraction-round1.md)

Current meaning:

- neurologic FIP is not just ordinary FIP with worse severity
- this branch changes dose, monitoring, relapse, and evidence interpretation
- it should not be allowed to backflow into the ordinary baseline claim

Claim-fit:

- neurologic FIP treatment support
- high-complexity branch extension
- dose / monitoring / relapse-aware framing

Claim style to avoid:

- using neurologic rescue evidence as ordinary baseline treatment language
- using baseline GS evidence as if it already covers severe neurologic and ocular disease
- collapsing the branch into a generic `FIP antiviral works` sentence

Current internal read:

- strategically important, but not the cleanest first regulatory archetype
- likely belongs as an extension branch or later label-expansion question unless product evidence is narrowed first

## What We Can Say Now

1. FIP antiviral regulatory work should begin with baseline GS-441524 as the cleanest product archetype.
2. Remdesivir plus GS package logic is operationally important but less product-clean.
3. Neurologic FIP treatment should stay as an extension branch unless the claim is deliberately narrowed.
4. FDA, EMA, China, and UK route questions are all real, but none is decision-grade for FIP antiviral launch strategy inside the current vault.
5. The most useful next regulatory move is eligibility testing, not jurisdiction ranking.

## What We Should Not Say Now

- do not recommend a first jurisdiction
- do not call conditional approval, Article 23, or any UK route available for this product yet
- do not treat treatment efficacy as regulatory legitimacy
- do not collapse baseline GS, remdesivir package, and neurologic rescue into one route object
- do not reuse remission durability as cure-rate or market-readiness proof

## Decision Logic This Memo Supports

1. Define the product object first.
2. Narrow the claim before choosing the route.
3. Use the FDA conditional-approval eligibility test result as a bounded input, then test EMA Article 23 eligibility explicitly.
4. Treat China and UK as framework-usable but implementation-thin until additional official materials are ingested.
5. Keep regulatory claims below treatment claims until product-specific precedent or guidance is available.

## Most Important New Insight

The strongest current FIP treatment branch is not automatically the cleanest regulatory route.

The cleanest first route object is probably:

`baseline GS-441524 antiviral product for non-neurologic naturally occurring FIP`

That is a working archetype, not a route recommendation.

## What To Do Next

1. Use the baseline GS U.S. comparator boundary memo to check current approved-record and compounding-list status.
2. Create an EMA Article 23 eligibility check for the same baseline archetype.
3. Ingest China companion animal implementing notices before making China route claims stronger.
4. Map VMD route options against the baseline GS archetype before attempting UK route recommendation.
5. Keep remdesivir package and neurologic rescue as second-pass route objects.
