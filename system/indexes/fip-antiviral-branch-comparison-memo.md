# FIP Antiviral-Branch Comparison Memo

- Date: `2026-04-10`
- Scope: `src-fip-013`, `src-fip-016`, `src-fip-017`, `src-fip-019`, `src-fip-024`

This memo exists because the current FIP translation layer is now repeatedly being forced to answer a narrower question:

`how should experimental foundation, baseline efficacy, package logic, neurologic rescue, and durability be compared without flattening them into one antiviral story?`

## Core Takeaway

`the FIP antiviral branch is now clearer as a branch-comparison problem than as a one-line efficacy problem`

## Current Branches

### Branch 1: Experimental Antiviral Foundation

Main anchor:

- `src-fip-017`

Current meaning:

- antiviral plausibility is real
- this branch explains why the treatment layer has a true preclinical floor
- it should stay below natural-disease efficacy in practical authority

### Branch 2: Baseline Natural-Disease Efficacy

Main anchor:

- `src-fip-016`

Current meaning:

- this remains the central practical anchor of the whole antiviral branch
- it is the strongest current reusable treatment-facing layer
- it should not be rewritten as universal efficacy across all disease forms

### Branch 3: Real-World Package Logic

Main anchor:

- `src-fip-019`

Current meaning:

- practical treatment has already moved beyond one clean efficacy anchor
- this branch matters because field treatment packages are not the same as single-study baseline logic
- it remains broader and more practice-like, but still retrospective

### Branch 4: Neurologic Rescue / High-Complexity Treatment

Main anchor:

- `src-fip-024`

Current meaning:

- neurologic FIP is not just baseline treatment under a worse prognosis
- this branch changes dose, monitoring, relapse, and difficulty logic
- it should remain visible as a separate high-complexity branch

### Branch 5: Remission Durability

Main anchor:

- `src-fip-013`

Current meaning:

- this branch extends the treatment map beyond initial response
- it sharpens post-treatment meaning
- it should not be promoted into a whole-branch cure-rate estimate

## What This Comparison Clarifies

- `baseline natural-disease efficacy` remains the practical leader
- `experimental foundation` stabilizes the lower layer, but does not lead clinical hierarchy
- `real-world package logic` adds operational realism, not clean superiority proof
- `neurologic rescue` is architecturally decisive because it creates a true branch shift
- `remission durability` matters after response, not in place of baseline efficacy

## What The Module Can Already Say

- FIP treatment transformation is real, but it is not one bucket
- the antiviral branch now contains at least five non-equivalent branches
- the branch is strong enough for comparison-layer compression even though final protocol hierarchy is still unresolved
- the ordinary compiled default should still lead with baseline natural-disease efficacy, not with the most dramatic rescue branch

## What The Module Should Not Yet Say

- there is still no safe flat protocol ranking across all branches
- experimental antiviral support should not replace natural-disease treatment authority
- package logic should not be rewritten as definitive superiority
- neurologic rescue should not be treated as ordinary baseline treatment
- durability follow-up should not be reused as a whole-branch denominator
