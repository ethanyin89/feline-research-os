# FIP Neurologic Rescue Boundary Memo

- Date: `2026-04-11`
- Scope: `src-fip-016`, `src-fip-019`, `src-fip-024`

This memo exists to answer a narrower question than the treatment evidence memo or the antiviral-branch comparison memo:

`what exactly stops neurologic rescue from being read as ordinary baseline antiviral treatment with more intensity?`

That boundary is already implied across the vault, but it is still too scattered.

## Core Takeaway

`neurologic rescue is a true high-complexity treatment branch, not an ordinary baseline antiviral branch with a worse prognosis or a simple dose extension`

## What This Memo Is Narrower Than

This memo is narrower than:

- [FIP treatment evidence memo](fip-treatment-evidence-memo.md)
- [FIP antiviral-branch comparison memo](fip-antiviral-branch-comparison-memo.md)

Those pages explain the larger treatment map.

This page answers the tighter boundary question:

`what exactly is neurologic rescue allowed to mean, and what should it not be allowed to backflow into?`

## Current Boundary

### Layer 1: Ordinary Baseline Antiviral Treatment

- baseline natural-disease efficacy remains the practical leader
- this branch carries the main reusable treatment-facing anchor
- it should not be silently generalized into severe neurologic disease

### Layer 2: Real-World Package Logic

- real-world remdesivir-plus-GS treatment matters because practice has already moved beyond a single clean anchor
- this adds operational realism, not universal hierarchy reset
- package logic still does not erase form-sensitive treatment difficulty

### Layer 3: Neurologic Rescue

- neurologic disease changes treatment complexity, monitoring burden, relapse logic, and difficulty
- this branch is architecturally decisive because it marks a real treatment-category shift
- the branch should remain visible precisely because it is not just “more of the same treatment”

## What The Module Can Already Say

- ordinary baseline efficacy and neurologic rescue are not the same treatment layer
- neurologic rescue matters because disease form changes what treatment means operationally
- dramatic rescue language should remain below whole-branch hierarchy claims
- neurologic rescue can be clinically important without becoming the default denominator for antiviral efficacy

## What The Module Should Not Yet Say

- there is no safe claim that neurologic rescue represents ordinary baseline efficacy
- there is no safe claim that rescue success rewrites the baseline branch
- there is no safe claim that package logic or rescue logic has settled final protocol hierarchy
- there is no safe claim that neurologic rescue outcomes can be reused as a whole-branch cure estimate

## Short Operational Read

| Treatment layer | Current role | What it clarifies | What it must not do |
|---|---|---|---|
| Baseline efficacy | main practical anchor | shows real natural-disease treatment transformation | do not universalize across neurologic complexity |
| Real-world package logic | practice-facing realism | shows broader implementation shape | do not rewrite as clean superiority |
| Neurologic rescue | high-complexity branch shift | shows treatment difficulty changes with disease form | do not backflow into ordinary branch meaning |
