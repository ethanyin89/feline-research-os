# HCM Product-Type Separation Memo

- Date: `2026-04-11`
- Scope: first-pass product-type separation beneath the current HCM regulatory stop-sign page

This memo exists to answer a narrower question than the current HCM regulatory brief:

`if route comparison is still too early, what product-type branches already need to be separated before any later route work can become coherent?`

The current HCM regulatory page already implies the answer.

This page makes it explicit.

## Core Takeaway

`for HCM, the first regulatory move is not jurisdiction comparison but product-type separation: drug-like, biomarker-assay, and device-like branches should be kept distinct before any route-level thinking`

## What This Memo Is Narrower Than

This memo is narrower than:

- [regulatory brief](../../topics/hcm/regulatory-brief.md)
- [HCM frontier augmentation memo](hcm-frontier-augmentation-memo.md)

Those pages explain the current stop-sign boundary and the scientific frontier map.

This page answers the tighter regulatory-structure question:

`which future product classes are already visible enough that the vault should stop talking about HCM as one generic future asset?`

## Current Product-Type Read

### 1. Drug-Like Branch

- anchored by targeted-therapy and broader treatment-comparison logic
- closest to a future medicine-style translational asset
- still too thin for route work, but already structurally distinct from assay or device logic

### 2. Biomarker-Assay Branch

- anchored by biomarker use-case and frontier-marker logic
- belongs closer to stratification, support, or bounded assay positioning than to therapeutic route reasoning
- should not be collapsed into drug-like regulatory language

### 3. Device-Like / Computational Branch

- anchored by AI augmentation logic
- belongs closer to augmentation, decision support, or device-like classification pressure than to assay or drug logic
- should not be collapsed into either treatment positioning or routine biomarker authority

## Why These Branches Must Separate Early

The current HCM module already knows enough to say:

- these three branches solve different jobs
- they imply different evidence packages
- they would likely face different regulatory questions later

If the vault waits too long to separate them, later route language will start from a false premise:

`HCM has one future product story`

It does not.

## What The Module Can Already Say

- future HCM regulatory work should begin with product-type separation, not jurisdiction ranking
- drug-like, assay-like, and device-like branches are already visible enough to deserve separate internal handling
- scientific richness in HCM is arriving earlier than regulatory readiness
- current pages support scientific mapping of these branches much earlier than route recommendation

## What The Module Should Not Yet Say

- there is no safe jurisdiction comparison yet for any of these branches
- there is no safe claim that one of the three product types is already the best regulatory target
- AI augmentation should not be mistaken for a ready-made device route claim
- biomarker frontier should not be mistaken for a ready-made assay product story

## Short Operational Read

| Product type | Current source of visibility | Why it must stay separate |
|---|---|---|
| drug-like | targeted therapy and treatment comparison | future medicine-style logic is different from assay or device logic |
| biomarker-assay | biomarker use-case and frontier-marker branch | stratification/support logic is not the same as treatment logic |
| device-like / computational | AI augmentation branch | augmentation and software-like pressure are not the same as assay or drug positioning |
