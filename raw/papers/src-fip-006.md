## Clinicopathology-Branch Role

This paper gives the FIP module a concrete recognition anchor. Mechanism papers explain how disease can emerge, and treatment papers explain why modern FIP is no longer only fatal-disease framing, but clinicians still need a practical layer describing what suspected FIP looks like in real cases.

The 42-case Sydney series is best used for that operational middle layer. It should help shape suspicion patterns, especially when paired with the Taiwan clinicopathology-and-staging paper (`src-fip-015`). The value is descriptive: repeated clinicopathologic patterns can improve recognition architecture even when they are not definitive.

The safe compiled rule is: clinicopathology should lead suspicion-building before molecular strengthening, but clinicopathology is not diagnostic certainty. This source sits below structured staging and above narrow assay sources in the recognition stack. It should be used to preserve effusive/non-effusive pattern awareness and older-cohort caveats, not to define modern treated-population outcomes.

For wiki reuse, this source should provide the `what does suspected FIP look like in cases` layer. It is especially useful because it is older and descriptive: that limits outcome claims, but it also keeps the recognition branch grounded in real-world case patterning rather than assay-first reasoning. In a Karpathy-style page, it should become one of the examples used to teach why routine clinicopathology starts the workup even when no single routine variable is decisive.

The relationship to `src-fip-015` should be hierarchical rather than competitive. `src-fip-015` is the stronger staging-oriented source; `src-fip-006` supplies an earlier Sydney cohort that can show whether the same broad recognition architecture repeats across geography and time. The future memo should use both sources to separate effusive and non-effusive suspicion patterns, while keeping the language below certainty. The card should not be used for prevalence, modern antiviral-era survival, or test-performance claims.

This also protects the diagnostic module from a common error: jumping straight from risk context to molecular testing. The intended sequence is risk context first, clinicopathologic form awareness second, then targeted confirmatory or supportive tests. This source belongs in that second layer.

As a source card, its main reusable asset is sequence discipline rather than a single headline finding.
